Sale View Customizations

Overview

This documentation covers the customizations made to the Sale (Pro-forma Invoice) views in Odoo. These changes enhance the sale order experience, improve workflow, and provide additional information and controls for users managing sales and quotations.

Key Customizations

  • Tree view: Replace create_date with date_order for better clarity on order timing.
  • Form view: Add an "Archived" ribbon and is_expired field for expired orders.
  • Form view: Hide the confirm buttons to control order confirmation workflow.
  • Form view: Add a new SOV (Statement of Value) page tab.
  • Form view: Add price_total field to order lines for enhanced financial visibility.
  • Menu and action renaming: Change all relevant menu and action names to "Pro-forma Invoice" for consistency.

Major Features

  • Clearer order date tracking in list/tree views.
  • Visual indication of expired/archived orders with a ribbon.
  • Controlled order confirmation process by hiding confirm buttons.
  • Additional SOV tab for custom business processes.
  • Improved financial overview with total price on order lines.
  • Consistent terminology across menus and actions for Pro-forma Invoices.

Use Case Scenarios

  • Sales teams can quickly identify and manage expired or archived orders.
  • Finance and management can see total order values directly in the order line list.
  • Users benefit from a streamlined and consistent Pro-forma Invoice workflow.
  • Custom SOV tab supports additional documentation or approval steps.

Technical Implementation

  • XML inheritance and xpath expressions to modify standard Odoo views.
  • Widget usage for ribbons and field visibility control.
  • Menu and action renaming via ir.ui.menu and ir.actions.act_window records.

Benefits

  • Improved sales workflow and clarity for end users.
  • Better tracking and management of Pro-forma Invoices.
  • Enhanced financial visibility and reporting.
  • Consistent user experience across the sales module.
